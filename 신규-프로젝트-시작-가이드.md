# 신규 프로젝트 시작 가이드

이 저장소는 한 레포 안에서 여러 프로젝트를 **branch + worktree** 로 분리해 운영한다. 이 문서는 사용자가 새 프로젝트를 시작할 때의 표준 절차를 정리한다. 에이전트 자동화 절차의 정식 정의는 `CLAUDE.md` 의 "새 프로젝트 생성" 섹션에 있고, 여기서는 사람이 직접 따라할 수 있는 형태로 풀어쓴다.

## 핵심 원칙

- 메인 레포: `/home/jai/program-agent` (브랜치: `main`).
- 각 프로젝트: `main` 에서 분기한 `project/{id}` 브랜치 + `/home/jai/pa/{id}` worktree.
- Framework 공용 파일(`CLAUDE.md`, `agents/`, `hooks/`, `signal/{schema,registry,execution-log}`)은 **`main` 에서만** 수정한다.
- 프로젝트별 파일(`projects/{id}/`, `signal/{id}/`)은 해당 프로젝트 워크트리에서만 보이며, 브랜치 격리로 자동 분리된다.
- archive 디렉토리는 쓰지 않는다. 작업 안 하는 프로젝트는 worktree 만 제거하고 브랜치는 origin 에 남긴다.

## 절차

### 1. 프로젝트 ID 결정

- 영문 소문자 + 하이픈 (예: `page-guide`, `stock-backtest`).
- 사용자에게 한 번 확인 후 확정한다. 추후 rename 은 브랜치/worktree/registry/symlink/관련 설정을 모두 갱신해야 하므로 비용이 있다.

### 2. 빈 디렉토리 준비 (선택)

`/home/jai/pa/{id}` 가 이미 비어 있으면 그대로 사용해도 되고, 없으면 `git worktree add` 가 만들어준다.

### 3. 브랜치 + 워크트리 생성

`main` 워크트리(`/home/jai/program-agent`)의 체크아웃 상태를 건드리지 않는 형태로 한 번에 처리한다.

```bash
git -C /home/jai/program-agent worktree add -b project/{id} /home/jai/pa/{id} main
```

- `main` 에서 분기한 `project/{id}` 브랜치를 새로 만들고, 해당 경로에 워크트리로 체크아웃한다.
- 메인 워크트리에 uncommitted 변경이 있어도 영향 없다.

### 4. 초기 signal / projects 스캐폴드

새 워크트리에서 다음 디렉토리/파일을 만든다 (스키마는 `signal/schema.md` 참고):

```
projects/{id}/.gitkeep
signal/{id}/task-board.md       # 빈 헤더 테이블
signal/{id}/done-log.md         # # Done Log
signal/{id}/architecture.md     # 빈 템플릿
signal/{id}/coder-report.md     # 초기 상태 frontmatter
signal/{id}/tester-report.md    # 초기 상태 frontmatter
signal/{id}/reviewer-report.md  # 초기 상태 frontmatter
```

모바일 앱 프로젝트라면 `signal/{id}/app-porter-report.md` 도 추가.

```bash
cd /home/jai/pa/{id}
git add projects/{id} signal/{id}
git commit -m "project/{id}: initial scaffold"
```

push 는 다른 머신에서도 worktree 를 띄울 계획이 있을 때만 한다 (`git push -u origin project/{id}`).

### 5. registry 등록 (main 에서)

`/home/jai/program-agent` 워크트리에서 `signal/registry.md` 의 활성 프로젝트 표에 새 행을 추가한다. `main` 의 다른 uncommitted 변경에 영향이 없도록 **파일 단위로** 스테이징한다.

```bash
cd /home/jai/program-agent
# 편집 후
git add signal/registry.md
git commit -m "main: registry 에 {id} 프로젝트 추가"
```

### 6. Claude Code 세션 히스토리 심볼릭 링크 (선택)

`~/.claude/projects/` 는 cwd 별로 세션 로그를 분리 저장한다. 같은 레포의 worktree 들이 같은 세션 풀을 공유하도록 메인 디렉토리를 가리키는 심볼릭 링크를 건다.

```bash
ln -sfn /home/jai/.claude/projects/-home-jai-program-agent \
        /home/jai/.claude/projects/-home-jai-pa-{id}
```

(이미 존재하면 `-sfn` 으로 안전하게 갱신.)

### 7. 새 세션 시작

```bash
cd /home/jai/pa/{id} && claude
```

Manager 가 `CLAUDE.md` 의 오케스트레이션 루프 Step 1 부터 진행한다 — `signal/{id}/` 가 모두 초기 상태이므로 사용자에게 요구사항을 요청하는 단계로 들어간다.

## 일시 중단 / 재개 / 종료

`CLAUDE.md` "프로젝트 lifecycle" 섹션 참고. 요약:

| 상태 | 표현 |
|------|------|
| 작업 중 | `project/{id}` 브랜치 + `/home/jai/pa/{id}` worktree 둘 다 존재 |
| 잠시 비활성 | 브랜치만 origin 에 남기고 worktree 제거 (`git worktree remove`) |
| 더 이상 작업 안 함 | 브랜치만 origin 에 보존, worktree 만들지 않음 |

재개 시:

```bash
git -C /home/jai/program-agent worktree add /home/jai/pa/{id} project/{id}
ln -sfn /home/jai/.claude/projects/-home-jai-program-agent \
        /home/jai/.claude/projects/-home-jai-pa-{id}
```

## Rename 시 주의

프로젝트 이름을 바꾸면 다음을 모두 갱신해야 한다:

- 브랜치 이름 (`git branch -m`, origin 도 갱신)
- 워크트리 경로 (`git worktree move` 또는 remove + add)
- `signal/registry.md` 의 ID/Branch/Worktree 칼럼
- `~/.claude/projects/` 심볼릭 링크
- 외부 설정에 절대 경로가 박혀 있다면 (예: 사용자 hook 의 스크립트 경로) 함께 수정
